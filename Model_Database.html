Berikut adalah analisis lengkap Model Database dan Relasi di project geonode-project-folurians:

🗄️ Model Database & Relasi

1. Hierarki Model Resource (GeoNode Core)

GeoNode menggunakan Polymorphic Model di mana semua resource (data spasial, dokumen, peta) mewarisi dari model dasar ResourceBase:

 

ResourceBase (PolymorphicModel)

├── Dataset → Layer spasial (vector, raster, tabular)

├── Document → File attachments (PDF, gambar, video, dll)

├── Map → Peta komposit (gabungan layer)

└── GeoApp → Aplikasi client generik

9. Model Custom: geonode_project

Project ini menambahkan 2 model custom:

 

A. SiteIdentity — Singleton (pk=1)

FieldTipeKeterangan
nama_kabupatenCharField(100)Nama kabupaten untuk branding
timezoneCharField(8)WIB / WITA / WIT
updatedDateTimeField(auto_now)Terakhir diupdate

Karakteristik:

- Selalu 1 baris saja (pk=1, dipaksa di save())

- Tidak berelasi dengan model GeoNode apapun

- Digunakan di template untuk menampilkan nama kabupaten dan zona waktu

 

B. ScreeningLog — Log WebGIS

FieldTipeRelasiKeterangan
nomor_regCharField(64), db_index—Nomor registrasi laporan
userForeignKey→ User (nullable)Pengguna (bisa anonim)
user_labelCharField(150)—Label tampilan ("Publik" default)
ipGenericIPAddressField—IP address
area_haFloatField—Luas area (hektar)
createdDateTimeField(auto_now_add)—Waktu log

Karakteristik:

- Relasi lemah ke User (nullable, SET_NULL)

- Tidak berelasi ke ResourceBase/Dataset/Document/Map

- Sengaja tidak pakai actstream karena WebGIS bisa diakses publik tanpa login

10. Activity Stream (django-actstream)

Model Action mencatat semua aktivitas di sistem:

FieldTipeKeterangan
actorGenericForeignKeyUser yang melakukan aksi
verbCharField"created", "updated", "deleted", "uploaded", dll
action_objectGenericForeignKeyObjek yang dikenai aksi (Dataset/Map/Document)
targetGenericForeignKeyTarget aksi (biasanya null)
timestampDateTimeFieldWaktu aksi
publicBooleanFieldVisibilitas publik
